# LLM Guide

Use this page as the first stop when an AI agent is reading or generating code for the Reserve DTF SDK.

## Package Boundary

- Use `@reserve-protocol/sdk` for scripts, bots, servers, CLIs, and non-React apps.
- Use `@reserve-protocol/react-sdk` for React apps that already use TanStack Query.
- React apps should prefer React SDK hooks over direct `useDtfSdk()` calls for UI reads.
- Both packages prepare transaction calls. They do not connect wallets, switch chains, send transactions, or own toast/confirmation UX.
- Do not import wallet-client constructors from the SDK. Browser apps use wagmi/connector wallet clients; scripts import wallet helpers directly from `viem`.
- `sdk.yield` covers Yield DTF (formerly RToken) reads, issuance, and staking on chains 1 and 8453. Governance/auctions/APY are not implemented yet.

## Canonical Example DTF

Use this real Base Index DTF in examples instead of the zero address:

```ts
const chainId = 8453;
const address = "0x4da9a0f397db1397902070f93a4d6ddbc0e0e6e8";
```

That address is LCAP from `indexDtfCatalog`.

## API Shapes To Preserve

- Direct SDK namespace methods use object params when multiple values are required: `sdk.index.get({ address, chainId })`.
- DTF refs bind `{ address, chainId }` once: `const dtf = sdk.index.ref({ address, chainId })`.
- Ref methods stay flat where possible: `dtf.get()`, `dtf.getProposal(proposalId)`, `dtf.prepareMint(params)`.
- React hooks accept params first and TanStack Query options second: `useIndexDtfBasket({ address, chainId }, options)`.
- Query options accept `sdk` first, params second, options third: `indexDtfBasketQueryOptions(sdk, params, options)`.
- Hooks with required params accept `undefined` to disable the query while route params or wallet state load.

## Data Shapes That Agents Often Get Wrong

- Index DTF baskets are records: `Record<Address, IndexDtfBasketAsset>`, not `{ tokens: [...] }`.
- Use `Object.values(basket)` for basket assets and `Object.keys(basket)` for asset addresses.
- Proposal IDs are long decimal strings. Keep them as strings in routes and UI state.
- `getProposalVoterState` takes a proposal detail object from `getProposal`, not a proposal ID string.
- `getBidQuote` returns `{ sellAmount, bidAmount, price }`; pass `quote.bidAmount` into `prepareBid({ maxBuyAmount })`.
- Onchain token amounts, shares, auction IDs, and raw balances are `bigint`. Do not cast money to `number`.
- `getIssuanceState` does not return `requiredApprovals`; derive approvals from asset allowances and mint amounts.

## Caching Defaults (React SDK)

- Every query options builder applies a `staleTime` default: 10s for live data (prices, auctions, issuance quotes), 30s for governance and account state, 5min for static metadata (identity, brand, version, catalogs, proposal decodes).
- Override per query via the options param: `useIndexDtfPrice(params, { staleTime: 0 })`.
- The constants are exported: `LIVE_STALE_TIME`, `DEFAULT_STALE_TIME`, `STATIC_STALE_TIME`.

## Write And Transaction Rules

- SDK write helpers return `ContractCall` objects.
- Always pass `call.chainId` through to simulation and wallet layers when supported.
- Use SDK builders for versioned writes instead of hand-encoding ABI calldata.
- DTF settings builders often need `version: "5.0.0" | "6.0.0"`; read it with `sdk.index.getVersion({ address, chainId })`.
- Raw version-sensitive `prepare*` helpers should receive the target DTF version explicitly. Higher-level proposal builders may fetch the DTF version when omitted.
- Proposal action builders prepare vote, queue, execute, and cancel calls. They do not invalidate queries after a transaction.

## Yield DTF Rules

- Yield DTF = the protocol formerly called RToken. ABIs keep contract-true names (`rTokenAbi`, `stRsrAbi`, `facadeReadAbi`); the API uses Yield DTF naming.
- There is NO Reserve API for Yield DTFs. Prices and quotes are on-chain via the Facade: `sdk.yield.getPrice` returns `{ low, high, price }` from `FacadeRead.price`.
- Canonical example: eUSD on mainnet — `{ address: "0xA0d69E286B938e21CBf7E51D71F6A4c8918f482F", chainId: 1 }`.
- Issuance: quote first (`getIssuanceQuote`), approvals go to the Yield DTF token itself, then `prepareIssue`/`prepareIssuePlan`. Hourly throttles apply — check `getState().issuance.available`.
- Staking: RSR approval goes to the staking vault (`stToken`); `exchangeRate` is RSR per stRSR and only grows (except seizure events). Unstaking is a two-step queue: `prepareUnstake`, wait `unstakingDelay`, then `prepareWithdraw(account, endId)` where `endId = pendingUnstake.index + 1`.
- Subgraph BigDecimal fields (holder balances, USD values, exchange rates) are display-class `number`s; raw BigInt fields (transaction amounts, supplies, stake record `*Raw` fields) map to `Amount`. Transaction `type: "burn"` is a furnace melt, not a user redemption.
- `basketsNeeded / totalSupply` is the basket-units-per-token exchange rate; both are D18 `Amount`s.

## Renamed Or Removed APIs

- `getFull` / `getFullIndexDtf` / `useFullIndexDtf` / `fullIndexDtfQueryOptions` no longer exist; `get` / `getIndexDtf` / `useIndexDtf` / `indexDtfQueryOptions` are the one name for the full DTF view.
- `getTransactions` no longer accepts `dtfPriceUsd` and does not return `amountUsd`; multiply `amount.formatted` by a price from `getPrice` if a USD value is needed.

## Vote Lock And Snapshot Helpers

- `sdk.index.getVoteLockState({ address, chainId, account })` resolves the vault from the DTF; `sdk.index.getVoteLockVaultState({ stToken, chainId, account })` reads straight from a staking vault when no DTF is known. Both include `underlyingPrice` when the price API is up.
- `mergeIndexDtfProposalVotingSnapshot(proposal, snapshot)` is the canonical way to overlay a fresher voting snapshot onto a proposal detail while a proposal is active.

## ABIs

All contract ABIs are exported from the sdk root (and re-exported by react-sdk): `dtfIndexAbi` (+ `V1/V2/V4`), `folioArtifactAbi` (v6), governance and staking-vault ABIs, `timelockAbi`, `optimisticTimelockAbi`, `selectorRegistryAbi`, `unstakingManagerAbi`, deployer ABIs, `daoFeeRegistryAbi`, and the proposal decode set (`dtfIndexProposalAbi`, `dtfIndexProposalAbiCatalog`, `indexDtfProposalDecoderAbi`). Never copy ABIs into a consumer project.

## Public Surface Discovery

Start from these source files when docs and types disagree:

- `packages/sdk/src/index.ts` for exported SDK types, constants, builders, ABIs, and catalog exports.
- `packages/sdk/src/create-dtf-sdk.ts` for the root namespace shape.
- `packages/sdk/src/index-dtf/namespace.ts` and `packages/sdk/src/index-dtf/ref.ts` for Index DTF direct/ref APIs.
- `packages/sdk/src/types/index-dtf.ts` and `packages/sdk/src/types/governance.ts` for data shapes.
- `packages/react-sdk/src/index.ts` for exported React hooks and query options.
- `packages/react-sdk/src/query-keys.ts` for shared query-key helpers.
- `packages/dtf-catalog/src/index.ts` for raw catalog exports like `indexDtfs` and `yieldDtfs`; `packages/sdk/src/index.ts` re-exports them as `indexDtfCatalog` and `yieldDtfCatalog`.

# Reserve DTF SDK

The Reserve DTF SDK is the TypeScript interface for building with Reserve DTFs.

It has two public packages:

- `@reserve-protocol/sdk` for scripts, bots, servers, CLIs, and apps that want direct async functions and transaction call builders.
- `@reserve-protocol/react-sdk` for React apps that want TanStack Query hooks around the same SDK.

The current implemented product surface is Index DTFs. Yield DTF docs are included for protocol context, but the SDK yield namespace is only a placeholder today.

## What The SDK Does

- Discover and list Index DTFs.
- Read live basket, supply, price, governance, rebalance, auction, revenue, and vote-lock state.
- Build proposal payloads for basket, DTF settings, DAO settings, and governance settings.
- Prepare transaction calls for mint, redeem, voting, queueing, execution, cancellation, deployment, auctions, and vote-lock actions.
- Expose source-aware clients for Reserve API, viem/RPC, subgraphs, and explorer metadata.

## What The SDK Does Not Do

- It does not connect wallets.
- It does not switch chains.
- It does not send transactions.
- It does not own app toasts, transaction confirmation, or post-transaction refresh.
- It does not hide whether data came from RPC, Reserve API, subgraph, or explorer metadata.

## Supported Chains

| Product    | Chains                                                  |
| ---------- | ------------------------------------------------------- |
| Index DTFs | Ethereum mainnet `1`, Base `8453`, BSC `56`             |
| Yield DTFs | Ethereum mainnet `1`, Base `8453` protocol context only |

Arbitrum exists in some legacy Reserve surfaces, but it is not part of the current SDK Index DTF chain scope.
